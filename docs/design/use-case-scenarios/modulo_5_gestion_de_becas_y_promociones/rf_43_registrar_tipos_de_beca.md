# RF-43: Registrar tipos de beca

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández (Actualizado por Antigravity) |
| **Nombre** | RF-43: Registrar tipos de beca y promociones |
| **Actor** | Administrador |
| **Objetivo** | Crear y registrar en el catálogo del sistema los tipos de beca y promociones estacionales disponibles, permitiendo definir un porcentaje fijo (como el 30% exclusivo para Beca de Hermanos) o una matriz de porcentajes dinámicos basados en Promoción × Nivel Educativo × Grado, para su uso estructurado. |
| **Flujo Principal** | 1. El Administrador ingresa al Panel de Configuración de Becas y Promociones.<br>2. Selecciona la opción "Registrar nueva Promoción o Beca".<br>3. El sistema despliega un formulario solicitando el nombre (ej. "Beca de Hermanos", "Marzo-Junio") y el tipo de estructura (Porcentaje Fijo o Matriz de Porcentajes).<br>4. Si es Porcentaje Fijo, el Administrador ingresa el porcentaje (ej. 30%).<br>5. Si es Matriz de Porcentajes, el sistema despliega una cuadrícula dinámica con las combinaciones de Nivel Educativo (Preescolar, Primaria, Secundaria, Bachillerato) y Grado/Semestre correspondientes.<br>6. El Administrador introduce los porcentajes específicos para cada celda de la matriz (ej. Preescolar 1º: 30%, Preescolar 2º: 25%, Primaria: 18%).<br>7. El Administrador hace clic en "Guardar".<br>8. El sistema valida que todos los campos obligatorios estén completos y los porcentajes estén entre 0 y 100.<br>9. El sistema almacena la promoción/beca en el catálogo y muestra un mensaje de éxito. |
| **Flujo Alterno** | **Flujo alterno A — Campos vacíos o porcentajes inválidos**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema detecta valores fuera de rango o campos requeridos vacíos.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema resalta las celdas o campos en rojo y solicita la corrección.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El Administrador corrige los datos y vuelve a guardar.<br><br>**Flujo alterno B — Promoción duplicada**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema detecta que el nombre de la promoción ya existe.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema indica el error y no permite duplicar el registro. |
| **Precondiciones** | El Administrador debe tener una sesión activa. |
| **Postcondiciones** | La promoción o beca queda almacenada y disponible como opción de asignación en los procesos de inscripción activa. |
| **Reglas de negocio involucradas** | • La beca de hermanos tiene un valor invariable de 30% de descuento.<br>• Las promociones de inscripción se definen mediante una matriz Promoción × Nivel Educativo × Grado.<br>• No se permiten registrar porcentajes negativos o mayores a 100.<br>• Se restringe la asignación a una única promoción/beca activa por alumno (regla de no doble beca). |
