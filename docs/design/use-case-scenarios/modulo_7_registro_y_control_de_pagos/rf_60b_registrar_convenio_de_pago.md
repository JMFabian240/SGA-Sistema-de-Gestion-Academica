# RF-60b: Registrar un convenio de pago

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Antigravity |
| **Nombre** | RF-60b: Registrar un convenio de pago |
| **Actor** | Administrador o Gestor Administrativo |
| **Objetivo** | Permitir al personal registrar un convenio de pago por escrito firmado por un padre de familia/tutor con deudas acumuladas, ingresando un monto consolidado y una fecha compromiso de pago, para detener temporalmente penalizaciones automáticas y formalizar el acuerdo en el sistema. |
| **Flujo Principal** | 1. El usuario accede al perfil del Tutor o del Alumno.<br>2. Selecciona la opción de "Registrar Convenio de Pago".<br>3. El sistema muestra la lista de adeudos pendientes seleccionables y el saldo deudor total consolidado.<br>4. El usuario selecciona los adeudos que serán cubiertos por el convenio.<br>5. El usuario ingresa la fecha compromiso de pago y de forma opcional un comentario o motivo.<br>6. El usuario confirma el registro del convenio.<br>7. El sistema genera el registro del convenio vinculándolo al Tutor y a los adeudos seleccionados.<br>8. El sistema actualiza el estado de esos adeudos específicos agregando una etiqueta visual de "Convenio Activo" y congela la aplicación de nuevos recargos por mora en ellos.<br>9. El sistema muestra un mensaje de éxito y permite imprimir la ficha del convenio para firma física. |
| **Flujo Alterno** | **Flujo alterno A — Convenio vencido sin pago**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. Transcurre la fecha compromiso del convenio sin que se registre el pago total.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema (proceso diario) detecta el convenio como vencido.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema retira la protección de congelación de recargos.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema actualiza el estatus a "Convenio Vencido" y reactiva los recargos acumulados correspondientes. |
| **Precondiciones** | 1. El usuario (Administrador/Gestor) debe estar autenticado.<br>2. El alumno/tutor debe tener deudas vencidas registradas en el sistema.<br>3. El usuario debe poseer privilegios de escritura. |
| **Postcondiciones** | • Se almacena el registro de `ConvenioPago` asociado a los adeudos correspondientes.<br>• Se congela temporalmente la generación de recargos automáticos para los adeudos en convenio hasta la fecha compromiso.<br>• El historial del tutor refleja el convenio activo. |
| **Reglas de negocio involucradas** | • Solo se puede registrar un convenio de pago activo a la vez por tutor.<br>• La fecha compromiso debe ser posterior a la fecha actual.<br>• Los adeudos asociados al convenio no acumularán el recargo de $400.00 MXN hasta el día posterior a la fecha compromiso.<br>• El convenio se cancela o liquida automáticamente cuando la suma de los abonos aplicados a los adeudos relacionados llegue al total del convenio. |
