# RF-57: Asignar plan de pago e inscripción express

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Antigravity |
| **Nombre** | RF-57: Asignar plan de pago e inscripción express |
| **Actor** | Administrador o Gestor Administrativo |
| **Objetivo** | Inscribir de forma rápida a un alumno en el ciclo escolar activo asignándole un grupo y un plan de pagos (10 o 12 meses), para generar automáticamente su calendario de adeudos correspondiente en la base de datos. |
| **Flujo Principal** | 1. El usuario accede al perfil de un Alumno que tiene estatus "Activo" o "Transición Pendiente".<br>2. Hace clic en el botón de "Inscribir a Ciclo Escolar".<br>3. El sistema despliega el formulario de inscripción con los campos: Ciclo Escolar, Plan de Pagos, Grupo Académico y Descuento/Beca Directo (opcional).<br>4. El usuario selecciona el Ciclo Escolar activo (ej. 2026-2027) y el Grupo Académico.<br>5. El usuario selecciona el Plan de Pagos (ej. "Plan 12 Meses" o "Plan 10 Meses").<br>6. El usuario hace clic en "Guardar Inscripción".<br>7. El sistema valida los datos y registra la relación `InscripcionCiclo`.<br>8. El sistema calcula y genera automáticamente en la tabla `CalendarioPago` las mensualidades correspondientes al plan:<br>&nbsp;&nbsp;&nbsp;&nbsp;a) Si seleccionó **Plan 10 Meses**: Genera 10 adeudos por el monto estándar de la colegiatura, sin cobros dobles.<br>&nbsp;&nbsp;&nbsp;&nbsp;b) Si seleccionó **Plan 12 Meses**: Genera las colegiaturas regulares y en el adeudo de diciembre calcula la cuota doble (que cubre la mensualidad de diciembre y de julio de vacaciones), dejando el mes de julio exento de cobro en el calendario.<br>9. El sistema muestra un mensaje de éxito y redirige a la ficha del alumno con sus adeudos listados. |
| **Flujo Alterno** | **Flujo alterno A — El alumno ya está inscrito en el ciclo**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 7, el sistema detecta un registro previo de inscripción activo para el alumno en el mismo ciclo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: "El alumno ya se encuentra inscrito en este ciclo escolar" y bloquea la duplicación. |
| **Precondiciones** | 1. El usuario (Administrador/Gestor) debe estar autenticado.<br>2. El alumno debe estar registrado en el sistema.<br>3. Deben existir tarifas base configuradas para el ciclo y nivel escolar seleccionados. |
| **Postcondiciones** | • Se crea el registro `InscripcionCiclo`.<br>• Se insertan de forma atómica los registros de `CalendarioPago` (10 o 12 adeudos) para el alumno según su plan.<br>• El estado financiero del alumno cambia a "Al Corriente". |
| **Reglas de negocio involucradas** | • La inscripción express genera automáticamente las colegiaturas basadas en el plan elegido.<br>• El plan de 12 meses duplica el cargo de diciembre para cubrir diciembre y julio (vacaciones), mientras que en julio no se cobra.<br>• El plan de 10 meses cobra 10 meses corridos y no genera cobro doble en diciembre ni cobros en periodos vacacionales exentos.<br>• Se hereda el porcentaje de beca/promoción asignado para descontarlo directamente de la base de cada mensualidad. |
